# Examples

This folder contains **reference patterns** for four common output
formats. Each file describes typical structure choices and shows how
a Curio DESIGN.md can be mapped to that format.

**These are not templates.** The canonical rules for using any Curio
design live in the top-level `../SKILL.md`. Read those first.

## When to look here

- ✅ You're composing a format you've never done from a Curio design
  AND this folder has an example for it → glance for structure ideas
- ✅ The user explicitly says "use the deck example as a starting point"
- ❌ Don't read these by default. The Universal Rules in `SKILL.md`
  are enough for most outputs.

## What's here

| File | Format | Typical dimensions |
|---|---|---|
| `deck.md` | Slide deck (PPT / pitch / keynote) | 1280 × 720 (16:9) |
| `landing.md` | Marketing landing page | Responsive, max 1280px |
| `poster.md` | Single-page poster | 1080 × 1350 default, plus 5 size presets |
| `card-social.md` | XHS / Tweet / IG carousel | 1080 × 1440 (XHS), 1200 × 675 (Twitter), etc. |

## What's NOT here, but you can still make

The four files above cover common formats. You are not limited to
these. If the user asks for:

- A report / long-form document
- An email template
- A newsletter
- A dashboard
- A lock-screen widget
- A resume
- A mobile app screen mockup
- Any other HTML deliverable

— make it. Follow the Universal Rules in `../SKILL.md`. Use your
judgment for structure. The design's tokens give you the visual
language; the user's intent gives you the format.

## Don't pad to match an example

A user asking for "a 4-slide product summary" doesn't need a
10-slide pitch sequence. A user asking for "a one-section landing
page with just a hero and signup" doesn't need 6 sections. Match
the actual ask.
