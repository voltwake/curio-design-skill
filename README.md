**Curio** is a curated library of design systems — Bauhaus, Stripe,
Muji, Memphis, vaporwave, Hokusai's ukiyo-e, Art Deco, and ~500
others. Each system is fully tokenized (colors, typography, spacing,
shadows, components) **and** comes with a cultural origin story plus
ready-to-use CSS techniques.

**This skill** gives your AI agent everything it needs to apply any
Curio design to any HTML output — slide decks, landing pages, posters,
reports, social cards, emails, dashboards, or anything else you
can render in HTML. You ask, the AI composes.

## Install

### Claude Code

```bash
git clone https://github.com/voltwake/curio-design-skill.git ~/.claude/skills/curio
```

Then in any Claude Code session: `"use Curio Bauhaus to make a 10-slide
pitch deck"` — the skill auto-triggers on `curio` / `用 Curio` / design
name / format keywords.

### Cursor / Codex / Gemini / OpenCode / Qwen / Aider

Clone anywhere your AI agent has read access, then point your
agent's "skills" or "rules" path at the cloned directory. The
top-level `SKILL.md` is the canonical entry point;
`CLAUDE.md` and `AGENTS.md` are 1-line redirects for
platform-specific conventions.

### Use without installing

Every share link from [designbycurio.com](https://designbycurio.com)
returns a complete `DESIGN.md`. Hand it to any AI agent along with
"please make a {deck|landing|poster|carousel|report|email} from this
design" and you're set — but the AI agent will produce better output
if it also has this skill's `SKILL.md` (which contains the universal
rules every output must honor).

## How it works

The skill is intentionally simple:

```
   ┌─────────────────────────────────────┐
   │  designs/<id>.md                    │   ← The visual language
   │                                     │     (colors, typography,
   │  tokens + cultural context +        │     spacing, shadows,
   │  CSS techniques + do's & don'ts     │     components, etc.)
   └────────────────┬────────────────────┘
                    │
                    │  +
                    ▼
   ┌─────────────────────────────────────┐
   │  SKILL.md                           │   ← How to read a DESIGN.md
   │                                     │     + universal rules for
   │  reading guide + universal rules    │     every output (layout,
   │  + workflow                         │     density, color, hier-
   │                                     │     archy, personality,
   │                                     │     defensive techniques)
   └────────────────┬────────────────────┘
                    │
                    ▼
            beautiful HTML output
        (whatever format you asked for)
```

**The skill does not dictate output format.** You ask for a deck and
get a deck. You ask for a newsletter and get a newsletter. You ask
for an email template and get an email template. Format follows
your ask; visual language follows the chosen design.

## Quick examples

```
"Use Curio Bauhaus to make a 10-slide pitch deck for my climate startup"
→ reads designs/bauhaus-weimar.md + SKILL.md
→ outputs HTML deck — red/blue/yellow primaries, Josefin Sans lowercase
  headings, hard-offset shadows, 0-radius corners, primary-color block
  section dividers, cream paper background

"Make a 小红书 carousel about 5 startup lessons, in Memphis style"
→ reads designs/memphis-sottsass-1981.md + SKILL.md
→ outputs N cards (cover + 5 idea + closing) on 1080×1440 canvas,
  with squiggle borders, candy colors, asymmetric layout

"Make an email newsletter about our product launch, in Stripe-2024 style"
→ reads designs/stripe-2024.md + SKILL.md
→ outputs HTML email — indigo accent, breathable whitespace, confident
  typography. No "method" required — the AI composes the format.

"Make a Bauhaus poster for my design talk"
→ reads designs/bauhaus-weimar.md + SKILL.md  
→ AI may glance at examples/poster.md for composition presets, then
  composes a real Bauhaus poster (geometric grid, primary colors,
  Kandinsky shape motifs)
```

## Curio philosophy

> *"No more designing. Pick a curio. AI dresses the rest."*

Curio is built on the conviction that good design should be a
**library**, not a **service**. Every cultural moment that produced
strong visual language — Bauhaus, Art Deco, ukiyo-e, vaporwave, brand
systems from Stripe to Discord — deserves to be **portable**: usable
by anyone, applied anywhere, with no learning curve.

If you've ever wanted to make a "Bauhaus deck" or a "Muji landing
page" but couldn't because you're not a designer — that's the gap
Curio fills.
